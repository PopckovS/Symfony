## SQL Хранимые процедуры
Хранимые процедуры - набор команд SQL для выполнения конкретных
операций

Преимущества:

- Для упрошщения сложных запросов
- Повторное исопльзование уже написанных строк кода на SQL
- ХП хранятся в скомпилированном виде, и выполняются быстро
- ХП абстрагируя от конкретной реализации может дать гибкость

Простота, Безопасность, Скорость.

Недостатки:

- Синтаксис сильно зависит от СУБД
- Сложны в написании

### Запуск хранимых процедур
Запускаются хранимые процедуры при помощи кода EXECUTE. EXECUTE - 
принимает название запускаемой хранимой процедуры и аргументы для ее
использования, в этом отношении ХП похожа на простую функцию.


    EXECUTE ADDNewProduct(
                    'JTS01',
                    'Stuff',
                     6.49,
                    'Plush .....'
    )

Это хранимая процедура, она принимает на вход ряд параметров, это
которые будут вставлены в запрос что хранится в хранмой процедуре.

Также в самой таблице есть поле первичного ключа prod_id но его мы 
не передаем,бо в самомо запрсе хранимой процедуры, это значение 
не передается для создания автогенерируемого поля первичного ключа
что позволяет недопустить ошибки.
